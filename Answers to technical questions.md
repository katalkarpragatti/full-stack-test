2. How would you track down a performance issue in production? Have you ever had to do this?

When investigating a production performance issue, I typically follow a structured approach:

Gather Information
Identify the symptoms, affected users, and time period.

Analyze Application Metrics
Check CPU, memory, disk I/O, and network utilization.
Review application response times and error rates.

Review Logs
Inspect application logs for exceptions, slow requests, or unusual patterns.
Correlate logs with the time when the issue occurred.

Profile Database Performance
Identify slow queries using database logs or query analysis tools.
Review indexes, execution plans, and query efficiency.

Reproduce and Validate
Attempt to reproduce the issue in a staging environment.
Apply fixes and validate improvements before deployment.
